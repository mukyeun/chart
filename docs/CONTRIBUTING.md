# 기여 가이드

## 개발 환경 설정
1. 저장소 포크
2. 로컬 개발 환경 설정
3. 브랜치 생성

## 코드 스타일
- ESLint 규칙 준수
- Prettier 설정 사용
- 컴포넌트 구조화 방침

## 커밋 메시지 규칙
- feat: 새로운 기능
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 포맷팅
- refactor: 코드 리팩토링
- test: 테스트 코드
- chore: 기타 변경사항

## Pull Request 프로세스
1. 기능 브랜치 생성
2. 코드 작성 및 테스트
3. PR 템플릿 작성
4. 리뷰 요청
